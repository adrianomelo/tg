Testing

This distribution contains a number of unit tests for the OWL API. JUnit tests are provided to test the basic API and some of the applications built on top of the API. Some of the tests use the OWL Test Cases as source material. The property tests.manifests controls the list of test sources that are used.

Directory resources contains a number of small OWL ontologies for testing individual language constructs. It also contains a list of URIs of manifests for OWL Tests.

Running the Tests

Tests for each module can be compiled and run using:

ant run.test

Some of these (for instance the build tests of the validator and consistency checkers) make use of the OWL Test Cases. Network connectivity is (of course) required to run these tests.

Test results for each module can be viewed in the directory build/module/test.results.

uk.ac.man.cs.img.owl.inference.ConsistencyCheckerBuildTest
This will run a consistency checker against the OWL consistency test tests. Each test is run twice:
  1. once in lax mode, where we consider a response of UNKNOWN to be acceptable
  2. once in strict mode, where a response of UNKNOWN is considered a failure.
The current implementation of the consistency checker is a basic stub that simple returns UNKNOWN as the result of any test. As a result, it will fail the strict tests and pass the lax tests.
